# Contributing to TelegramKt

Thanks for your interest! 🚀

## Quick Start

1. **Fork** the repository
2. **Clone** your fork: `git clone https://github.com/telegram-kt/telegram-kt.git`
3. **Build**: `./gradlew build`
4. **Branch**: `git checkout -b feature/my-feature`
5. **Commit**: `git commit -m "Add: feature description"`
6. **Push**: `git push origin feature/my-feature`
7. **Open PR** on GitHub

## Code Style

- Official [Kotlin Coding Conventions](https://kotlinlang.org/docs/coding-conventions.html)
- 4 spaces indentation
- Max line length: 120 characters

## What to Contribute

- [ ] API methods coverage (see `TelegramApi.kt`)
- [ ] More examples in `/examples`
- [ ] Documentation improvements
- [ ] Bug fixes
- [ ] Tests (we need them!)

## Questions?

Open an [Issue](https://github.com/telegram-kt/telegram-kt/issues) or DM in Telegram.

## 🏆 Join the Core Team

Enjoying the project? Want to help regularly?

**Request access to the organization!**

If you're making consistent contributions and want to be part of long-term development:

1. Send a message to <a href="mailto:telegramkt.dev@proton.me?subject=Application%20for%20contribution&body=GitHub%20username:%0A%0AWhat%20I'd%20like%20to%20work%20on:%0A%0AWhy%20I'm%20interested:">telegramkt.dev@proton.me</a> with:
    - Your GitHub username
    - What you'd like to work on
    - Why you're interested

2. We'll review and add you to the `telegram-kt` organization

**Benefits:**
- Direct push access (no forks)
- Input on architectural decisions
- Co-maintainer status for active contributors
- Recognition in README

Everyone starts with a PR — show us what you've got! 💪
